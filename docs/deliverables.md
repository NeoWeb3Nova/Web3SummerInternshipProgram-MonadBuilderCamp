# 交付物标准

## 总体要求

所有交付物应满足：
- **公开可验证**: 尽量使用公开链接、链上地址、GitHub Repo
- **可复用**: 文档清晰，他人能看懂并复现
- **真实**: 记录真实过程，包括失败与迭代
- **AI 协作透明**: 标注 AI 在每一步中的作用与人的判断

## 最终交付物清单

### 1. 链上产品 Demo
- 可访问的 Demo 链接（Vercel / Netlify / 其他）
- 支持的链/网络（ Monad Testnet 优先）
- 核心功能说明
- 测试账号/钱包说明

### 2. GitHub Repo
- 清晰的 README
- 代码结构合理
- 包含部署脚本/说明
- 开源协议
- Commit 历史反映真实开发过程

### 3. Build Log
- 记录 AI 使用场景
- 记录关键决策与技术选型
- 记录遇到的问题与解决方案
- 每日/每周更新

### 4. 研究 / 运营 / 技术材料
根据所选 Track：
- Tech: 部署文档 + 技术复盘
- Ops: Campaign 方案 + 增长数据 + 复盘
- Research: 研究文章 + 生态地图 + Product Thesis

### 5. Portfolio Page
- 一页式项目展示
- 包含项目背景、角色、成果、链接
- 适配简历与面试场景

### 6. Hackathon 提交包
- Pitch Deck
- Demo Video（建议 2-3 分钟）
- Final Presentation
- 团队介绍与分工

## README 检查清单

- [ ] 项目一句话介绍
- [ ] 背景/问题
- [ ] 解决方案
- [ ] 技术栈/工具
- [ ] 如何本地运行
- [ ] 如何部署
- [ ] 合约地址/测试网链接
- [ ] 团队/分工
- [ ] AI 协作说明
- [ ] 许可证
