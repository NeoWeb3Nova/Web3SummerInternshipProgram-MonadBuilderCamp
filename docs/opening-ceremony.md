## Box (Monad Foundation) 分享：Monad 技术与生态

### 为什么需要高吞吐量的区块链

- 以太坊理论 TPS 很低，无法支撑大规模应用
- 历史上每次量级产品（FT、DeFi Summer、NFT、BTC 铭文）都会带来交易量突增
- 瓶颈不在于应用多少，而在于基础设施能否支撑未来爆发
- 可类比：2015 年南山区六车道，早期车少时被质疑，但后期高楼平地起后完全不够用

### Monad 是什么

- 在不牺牲去中心化的前提下实现高性能
- 完全兼容 EVM，以太坊工具链（RPC、Foundry、Hardhat、MetaMask 等）可无缝迁移
- 主要数据：500M gas/s、出块时间 0.4s、持续 TPS 10,000、200 个验证者
- 目标：抬高链的性能上限，而不是提高门槛
- 希望家用电脑（RTX 3070/3090、32G 内存、2TB 硬盘）也能轻松运行节点

### 三项架构创新

1. **流水线化的共识与执行**
   - 大多数链是交错执行：执行 → 共识 → 执行 → 共识
   - Monad 将共识与执行并行化：在共识 n+2 区块的同时执行 n+1 区块
   - 共识阶段只排序交易，不执行；执行阶段才确定状态结果

2. **多线程乐观并行执行**
   - 假设交易之间独立无冲突
   - 从同一状态快照并行执行交易
   - 通过状态暂存区逐步比对，发现冲突则重新执行
   - 即使有重执行，总耗时也远低于完全串行执行

3. **MonadDB 状态访问**
   - 针对 SSD 架构优化的数据库
   - 支持并发读取/写入，最大化利用 SSD 并行能力
   - 这部分是性能提升最大的环节

### 共识层创新

- **MonadBFT**: 半成晶算法
- **Refined Block Propagation**: 高效传播区块，而不是一个人发给所有验证者
- **GIL Optimization**: 公平性/性能优化
- Research Track 可深入对比 Monad 与传统共识算法的差异
