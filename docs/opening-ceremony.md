## 实习计划目标

活动希望解决新人学习 Web3 时常见的几个问题：信息碎片化、文档/白皮书难懂、缺少系统路径、缺少真实项目实践。

学员将通过一个月左右的共学完成：

- 理解 Web3 基础知识和链上交互
- 完成第一笔链上交易
- 和小伙伴组队做出 demo / MVP
- 做出第一份可展示的作品集
- 认识第一批 Web3 建设者伙伴
- 获得实习、生态大使、项目孵化等后续机会

## Box 分享：Monad 技术与生态

### Box 背景

Box 来自 Monad Foundation，负责中国地区 DevRel（开发者关系）。他建议学员遇到问题时先问 AI，AI 解决不了再在群里 @ 他或直接联系。

### 为什么需要高吞吐区块链

Box 从历史增长曲线解释：NFT、DeFi Summer、Meme coin、BTC 铭文等每次链上热点都会带来交易量冲击。以太坊当前吞吐量不足以支撑未来全球级应用。

核心判断：瓶颈不在于“有没有新应用”，而在于底层链是否预先提供足够基础设施。就像早期建设六车道，看似浪费，但当高楼、企业、用户增长后反而不够用。

### Monad 的定位

Monad 希望在高性能和去中心化之间取得平衡：

- 不靠牺牲验证者数量来获得性能
- 不像强共识但低性能链那样牺牲吞吐
- 完全兼容 EVM，MetaMask、Foundry、Hardhat、RPC、DeFi 应用等可低成本迁移
- 目标是抬高链性能上限，而不是提高运行门槛
- 希望普通消费级设备也能跑节点，而不是依赖超级服务器

提到的核心数据：

- 500M gas/s
- 0.4 秒出块
- 持续 TPS 可达 10,000
- 约 200 个验证者

### 架构创新

Box 将 Monad 的创新分为执行层和共识层。

执行层三点：

1. **流水线化共识与执行**
   - 传统链常见模式是执行和共识交替进行
   - Monad 尝试让共识和执行并行：共识新区块时，同时执行已共识的前一区块
   - 共识阶段只负责交易排序和打包，不立即计算交易结果

2. **多线程乐观并行执行**
   - 乐观假设交易之间互不冲突
   - 多笔交易从同一状态快照并行执行
   - 执行后按顺序比对状态，发现冲突再重执行相关交易
   - 即使有少量重执行，也比完全串行执行更快

3. **MonadDB 状态访问**
   - 针对 SSD 架构优化数据库访问
   - 支持更高效的并发读写
   - 解决以太坊早期状态读取难以并行的问题

共识层三点：

- MonadBFT
- RaptorCast / 高效区块传播
- 延迟 / GIL 相关优化

Box 建议 Research 方向同学可以研究 Monad 共识与现有共识算法的差异。

### 开发者工具与生态

Box 提到 Monad 对 EVM 工具链兼容：Foundry、Hardhat、Ethers、Viem、MetaMask 等都可使用。

还提到：

- 官方文档和 AI 可读资料
- Monad skill，可帮助 agent 处理水龙头、合约开发、部署、验证、前端、索引器等
- x402 相关支持，用于 agent payment / 链上支付交互
- MCP / ERC-8004 等可进一步了解
- 钱包、索引器、预言机、账户抽象、RPC 等生态 provider 已有很多，不必重复造轮子
- 可本地 fork Monad 主网用于原型和协议联调

### 后续机会

- **AI Blueprint**: AI 应用加速计划，合作企业可能提供基础设施资源，例如 token / credits
- **Delta V**: 类 Gitcoin / 孵化器平台，定期更新项目动态，有机会获得曝光和 VC 关注
- **Monad Bless**: 与 OpenBuild 等合作举办的一天线下/短期黑客松，提供场地、WiFi、食物、导师、奖金和周边
- Monad 开发者社群：感兴趣可联系 Box 加入
