## 今日收获

1. 阅读真实开源项目应从 README 建立问题模型，再用 Docs、目录、Issues 与 PR 交叉验证，而不是直接从第一行源码开始。
2. Moss 用 Plan 的量化 `expects` 声明允许发生什么，再用 `simulate` 把实际 effects 与声明对账；零 warning 后仍需检查用户意图是否一致。
3. Maintainer 通过 ADR、Issue 标签、模板、CI 和小步 PR，把设计讨论、贡献入口与质量门槛编码进仓库。
4. MockVault Demo 与通用 ERC-4626 Interface Layer 的难度不同：前者验证实现形状，后者必须先解决实例化、verb 和 share / asset expects 等公共设计问题。
