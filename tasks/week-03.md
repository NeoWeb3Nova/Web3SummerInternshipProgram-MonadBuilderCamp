# Week 3: 跨轨协作与 Mini Demo

## 本周目标

跨轨组成小组，把想法推进成可展示的 prototype。

## Day-by-Day 任务

### Day 15: 组队与想法
- [ ] 找到至少 2 名不同 Track 的伙伴
- [ ] 确定团队角色分工
- [ ] 在 `submissions/week-03/` 创建 `team.md`

### Day 16: 问题定义
- [ ] 讨论并确定要解决的问题
- [ ] 验证问题是否适合在 Monad 上解决
- [ ] 输出 `problem-statement.md`

### Day 17: 方案设计
- [ ] 设计产品 / 研究 / 运营方案
- [ ] 明确核心功能与范围
- [ ] 制定 1 周开发计划

### Day 18: 开发/执行
- [ ] Tech: 搭建核心合约与前端框架
- [ ] Ops: 设计并执行增长动作
- [ ] Research: 收集论证并撰写分析

### Day 19: 集成与测试
- [ ] 集成各部分
- [ ] 进行联调测试
- [ ] 修复显著问题

### Day 20: Demo 准备
- [ ] 制作 Demo 演示流程
- [ ] 准备 3-5 分钟演讲
- [ ] 预演并收集反馈

### Day 21: Mini Demo
- [ ] 参与 Mini Demo Day
- [ ] 收集导师/同学反馈
- [ ] 修改并更新计划
