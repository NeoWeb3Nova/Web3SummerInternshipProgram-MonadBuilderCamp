> Week 2 的目标不是让所有人做同一种任务，而是帮助每位 Builder 在 Research / Ops / Dev 三条路径中选择一个主方向，完成一份可验证的 Proof of Work，并为 Week 3 组队准备清晰的角色定位。

## 第二周结束后，你将能够

- 🎯 找到适合自己的 Web3 职业方向（Research / Ops / Dev）

## DAY 1｜职业方向选择：我更适合做研究、运营还是开发？

### 🎯 学习目标

- 理解 Web3 团队中 Research / Ops / Dev 三类角色分别解决什么问题。
- 根据兴趣、能力、时间和 Week 3 组队目标，选择 Dev 为主职业方向。
- 建立 Week 2 Role Log，用于记录学习材料、Prompt、链接、截图、错误、判断变化和每日产出。
- 完成第一次 AI Collaboration Log，明确哪些工作可以交给 AI，哪些必须由人类判断和核查。

### 📖 学习内容

一个完整的 Web3 产品或项目不只需要开发者，通常需要三类 Builder 共同协作：

- **Research Builder**：发现问题，理解协议、产品、治理、生态机会和风险。
- **Ops Builder**：连接用户，设计社区运营、Space / 小型活动策划、传播执行和反馈复盘机制。
- **Dev Builder**：解决问题，把需求变成可运行的原型、工具、合约、脚本或 Agent 能力。

课堂练习：完成 Role Choice Card，回答：

1. 我选择哪个方向？
2. 为什么选择这个方向？
3. 本周最小可交付产出是什么？
4. Week 3 我可以在团队中承担什么角色？

## DAY 2｜路径启动：把职业方向变成一个可执行 Scope

### 🎯 学习目标

学习如何定义一个 AI-assisted MVP，并控制开发 Scope。

### 📖 学习内容

定义一个 AI-assisted Web3 Prototype。可以选择自己设计一个 Demo，也可以选择参与 Moss Open Source Challenge（推荐给有开发基础的同学）。写清 Scope、Architecture、Tech Stack、真实实现 / mock / 本周不做的边界。

### 📚 推荐资料

Monad Developer Docs、Cursor Docs、GitHub、Moss Repository（可选）。

### 🛠 今日任务

- **任务 15｜AI-assisted Dev Plan**：定义一个最小 Web3 原型，说明用户动作、技术组件、要读的文档、真实实现 / mock / 本周不做的边界。+20

### 📦 当日产出

AI-assisted Dev Plan + 技术组件清单。

## DAY 3｜路径深入：学习本方向真正的 Workflow

### 🎯 学习目标

学习 AI Coding Workflow，并理解开源项目的协作方式。

### 📖 学习内容

- 学习 AI 开发工作流：让 AI 读文档、拆任务、生成代码骨架、解释错误；人类负责运行、修改、验证和记录。
- 学习如何阅读开源项目：README、Issues、Pull Requests、Code Structure、Contributor Guide。
- 可选：阅读 Moss 项目结构，选择一个可能贡献的方向。

### 📚 推荐资料

Cursor Docs、GitHub Docs、GitHub Pull Request Guide、Moss Repository。

### 🛠 今日任务

- **任务 16｜文档到代码骨架**：用 AI 辅助阅读一个 Web3 / AI / 钱包 / 合约 / API 文档，生成代码骨架或技术方案，并记录人工修改。+30
- **Dev Bonus**：提交一个 Moss Issue、Documentation Draft 或 Draft PR。+10～20

### 📦 当日产出

- Code Skeleton / 技术方案 + AI Collaboration Log 初稿
- Issue / Draft PR / Documentation Draft（如有）

## DAY 5｜路径收束：整理成 Week 3 可组队 Portfolio

### 🎯 学习目标

- 把 Week 2 的学习、任务、证据和反思整理成可展示的 Portfolio Pack。
- 准备进入 Week 3 组队，明确自己能承担的角色和需要的队友。
- 将 AI Collaboration Log 与人工判断记录纳入作品集，避免只展示结果、不展示过程。

### 📖 学习内容

整理 Dev Portfolio Pack：Dev Plan、repo / demo、README、运行证据、AI Collaboration Log、Known Issues 和 Week 3 可贡献方向。

如果参与 Moss，可加入 Open Source Contribution：PR、Issue、Discussion、Documentation、Demo、Technical Blog 或 Tutorial。

### 📚 推荐资料

GitHub Profile / README Portfolio、Notion Portfolio 页面样例、Hackathon Demo 展示资料、Loom / 录屏工具、Notion / Google Sheet。

### 🛠 今日任务

- **任务 4｜Week 3 Role Statement**：用 100–300 字说明进入 Week 3 团队后能承担什么角色、需要什么队友、能提供什么 Proof of Work。+20
- **任务 18｜Dev Portfolio Pack**：汇总 Dev Plan、代码 / repo、README、运行证据、AI Collaboration Log、Known Issues、Week 3 开发角色。+50

- Dev Portfolio Pack
