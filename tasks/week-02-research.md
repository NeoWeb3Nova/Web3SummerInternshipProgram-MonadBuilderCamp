# Week 2｜Research Track：职业方向选择 + 研究能力训练

> Week 2 的目标不是让所有人做同一种任务，而是帮助每位 Builder 在 Research / Ops / Dev 三条路径中选择一个主方向，完成一份可验证的 Proof of Work，并为 Week 3 组队准备清晰的角色定位。

---

## 第二周结束后，你将能够

- 🎯 找到适合自己的 Web3 职业方向（Research / Ops / Dev）
- 🤖 学会使用 AI 协助研究、运营或开发，而不是只会聊天
- 📚 独立阅读 Web3 协议文档，并快速理解一个项目
- 🛠 完成一份属于自己的 Proof of Work（研究、运营方案或 Demo）
- 👥 带着作品集进入 Week 3，与其他 Builder 组队完成真实项目

## DAY 1｜职业方向选择：我更适合做研究、运营还是开发？

### 🎯 学习目标

- 理解 Web3 团队中 Research / Ops / Dev 三类角色分别解决什么问题。
- 根据兴趣、能力、时间和 Week 3 组队目标，选择 Research 为主职业方向。
- 建立 Week 2 Role Log，用于记录学习材料、Prompt、链接、截图、错误、判断变化和每日产出。
- 完成第一次 AI Collaboration Log，明确哪些工作可以交给 AI，哪些必须由人类判断和核查。

### 📖 学习内容

一个完整的 Web3 产品或项目不只需要开发者，通常需要三类 Builder 共同协作：

- **Research Builder**：发现问题，理解协议、产品、治理、生态机会和风险。
- **Ops Builder**：连接用户，设计社区运营、Space / 小型活动策划、传播执行和反馈复盘机制。
- **Dev Builder**：解决问题，把需求变成可运行的原型、工具、合约、脚本或 Agent 能力。

课堂练习：完成 Role Choice Card，回答：

1. 我选择哪个方向？
2. 为什么选择这个方向？
3. 本周最小可交付产出是什么？
4. Week 3 我可以在团队中承担什么角色？

## DAY 2｜路径启动：把职业方向变成一个可执行 Scope

### 🎯 学习目标

学习如何提出一个好的 Research Question，并明确资料边界。

### 📖 学习内容

Research Track 采用"两条线"：一个任务专门读 EIP / MIP，其他任务研究真实产品 / 协议。

- 从 DefiLlama Top TVL 协议、Uniswap / Morpho / Pendle 等代表性产品中选择研究对象。
- 写清研究问题、资料来源、产品分类、核心机制和想回答的用户 / 产品问题。
- 区分事实、数据、假设、推断和个人判断。

### 📚 推荐资料

DefiLlama / DefiLlama Protocols、Uniswap Docs、Morpho Docs、Pendle Docs、Aave Docs、Lido Docs、Monad Docs、Ethereum Docs、Messari Research、Monad Forum。

### 🛠 今日任务

- **任务 5｜Product Research Question Card**：从 DefiLlama Top TVL 协议、Uniswap / Morpho / Pendle 等代表性产品中选择一个研究对象，写清研究问题、资料边界和至少 5 个来源。+10

### 📦 当日产出

Research Question Card + 来源列表。

## DAY 3｜路径深入：学习本方向真正的 Workflow

### 🎯 学习目标

学习如何阅读一个 EIP / MIP，同时拆解一个真实产品 / 协议，而不是只摘抄概念。

### 📖 学习内容

- 阅读一个 EIP / MIP / ERC / 治理提案，训练理解底层机制和标准变化的能力。
- 同时阅读一个真实产品 / 协议，例如 Uniswap、Morpho、Pendle、Aave、Lido，或 DefiLlama Top TVL 产品。
- 拆解产品解决的问题、核心机制、用户角色、收入 / 激励、风险、竞争对象和产品启发。
- 训练"能否讲给新人听"的解释能力。

### 📚 推荐资料

DefiLlama Protocols、Uniswap Docs、Morpho Docs、Pendle Docs、Aave Docs、Lido Docs、Monad Forum、Ethereum Magicians、Ethereum EIPs、Messari Research。

### 🛠 今日任务

- **任务 6｜EIP / MIP Reading Card**：选择一个 EIP / MIP / ERC / 治理提案，写清背景问题、方案、影响对象、争议 / 风险和你的疑问。+20
- **任务 7｜Product / Protocol Reading Card**：选择 Uniswap / Morpho / Pendle / Aave / Lido 或 DefiLlama Top TVL 产品之一，用新人能懂的语言解释"它解决什么问题、核心机制是什么、怎么产生收益 / 激励、主要风险是什么"。+30

### 📦 当日产出

EIP / MIP Reading Card + Product / Protocol Reading Card。

## DAY 4｜路径产出：完成第一个 Proof of Work

### 🎯 学习目标

从提案 / 产品阅读走向自己的产品与市场判断。

### 📖 学习内容

写 Product-to-Market Brief。

- 说明一个真实产品 / 协议服务什么用户、解决什么市场问题，以及可能带来什么产品机会、风险或生态影响。
- 可以引用 EIP / MIP 作为底层机制背景，但主线应回到产品和市场判断。
- 必须包含资料来源、自己的判断和反方观点。

### 📚 推荐资料

DefiLlama、协议官方文档、Dune / Token Terminal（可选）、Messari Research、Paradigm Writing、Monad Forum、Ethereum Research。

### 🛠 今日任务

- **任务 8｜Product-to-Market Brief**：基于一个代表性 DeFi 产品 / 协议，说明它服务什么用户、解决什么市场问题、可能带来什么产品机会、生态影响或风险，包含资料来源和反方观点。+40

### 📦 当日产出

Product-to-Market Brief v1。

## DAY 5｜路径收束：整理成 Week 3 可组队 Portfolio

### 🎯 学习目标

- 把 Week 2 的学习、任务、证据和反思整理成可展示的 Portfolio Pack。
- 准备进入 Week 3 组队，明确自己能承担的角色和需要的队友。
- 将 AI Collaboration Log 与人工判断记录纳入作品集，避免只展示结果、不展示过程。

### 📖 学习内容

整理 Research Portfolio Pack：研究问题、资料来源、提案阅读卡、底层机制解释、产品判断、反方观点和 Week 3 可贡献方向。

### 📚 推荐资料

GitHub Profile / README Portfolio、Notion Portfolio 页面样例、Hackathon Demo 展示资料、Loom / 录屏工具。

### 🛠 今日任务

- **任务 4｜Week 3 Role Statement**：用 100–300 字说明进入 Week 3 团队后能承担什么角色、需要什么队友、能提供什么 Proof of Work。+20
- **任务 9｜Research Portfolio Pack**：汇总阅读卡、来源表、通俗解释、Product Brief、AI-assisted Research Log、Week 3 研究 / 产品角色。+50
