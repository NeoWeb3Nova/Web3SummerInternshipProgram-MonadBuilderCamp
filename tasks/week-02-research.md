# Week 2 Research Track: 研究、分析与叙事

## 本周目标

基于开营仪式和 Monad 技术分享，完成一个可公开展示的研究材料。研究方向可以是技术架构、产品机会、生态地图、DAO 治理或数据分析。

## 建议研究问题

从以下问题中选一个：

1. 为什么高吞吐区块链是未来链上应用的基础设施？
2. Monad 如何在 EVM 兼容和高性能之间取得平衡？
3. 流水线化共识与执行、乐观并行执行、MonadDB 分别解决什么问题？
4. Monad 与 Ethereum / Solana 的取舍有什么不同？
5. AI agent + x402 + Monad 可能催生什么产品？
6. Monad 新人开发者最容易卡在哪里？如何设计 onboarding？

## Day-by-Day 任务

### Day 8: 材料阅读

- [ ] 阅读 `docs/opening-ceremony.md`
- [ ] 阅读 `docs/monad-technical-notes.md`
- [ ] 阅读 Monad 官方文档 / Blog 至少 2 篇
- [ ] 整理关键术语：EVM、TPS、gas/s、BFT、并行执行、状态访问、RaptorCast

### Day 9: 研究问题与框架

- [ ] 确定一个研究问题
- [ ] 选择分析框架：技术架构 / 竞品对比 / 产品机会 / 生态地图 / 治理分析
- [ ] 在 `submissions/week-02-research/research-outline.md` 写大纲

### Day 10: 资料与案例收集

- [ ] 搜集 3-5 个相关项目/案例
- [ ] 记录每个案例的定位、用户、技术或生态关系
- [ ] 标注引用来源
- [ ] 使用 AI 整理材料时，记录 prompt 和人工筛选标准

### Day 11: 分析与论证

- [ ] 形成核心观点
- [ ] 写出支持证据和可能反驳
- [ ] 用表格或图示解释关键机制
- [ ] 输出 `analysis-notes.md`

### Day 12: 初稿撰写

- [ ] 撰写研究文章初稿
- [ ] 包含：问题、背景、机制解释、案例、结论、下一步
- [ ] 避免只复述资料，要写出自己的判断

### Day 13: 反馈与完善

- [ ] 发给 1-2 位同学/助教审阅
- [ ] 收集反馈
- [ ] 修改结构、论证和表达
- [ ] 制作可视化材料（生态地图 / 架构图 / 对比表）

### Day 14: 提交与复盘

- [ ] 最终定稿
- [ ] 输出可公开发布版本
- [ ] 写研究复盘：我学到了什么、下一步可做什么项目
