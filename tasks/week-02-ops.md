## DAY 1｜职业方向选择：我更适合做研究、运营还是开发？

### 🎯 学习目标

- 理解 Web3 团队中 Research / Ops / Dev 三类角色分别解决什么问题。
- 根据兴趣、能力、时间和 Week 3 组队目标，选择 Ops 为主职业方向。
- 建立 Week 2 Role Log，用于记录学习材料、Prompt、链接、截图、错误、判断变化和每日产出。
- 完成第一次 AI Collaboration Log，明确哪些工作可以交给 AI，哪些必须由人类判断和核查。

### 📖 学习内容

一个完整的 Web3 产品或项目不只需要开发者，通常需要三类 Builder 共同协作：

- **Research Builder**：发现问题，理解协议、产品、治理、生态机会和风险。
- **Ops Builder**：连接用户，设计社区运营、Space / 小型活动策划、传播执行和反馈复盘机制。
- **Dev Builder**：解决问题，把需求变成可运行的原型、工具、合约、脚本或 Agent 能力。

课堂练习：完成 Role Choice Card，回答：

1. 我选择哪个方向？
2. 为什么选择这个方向？
3. 本周最小可交付产出是什么？
4. Week 3 我可以在团队中承担什么角色？

## DAY 2｜路径启动：把职业方向变成一个可执行 Scope

### 🎯 学习目标

学习如何定义一场基础 Space / AMA / 小型线上活动的目标、受众、关键动作和执行边界。

### 📖 学习内容

Day 2 的 Ops 重点不是“做增长”，而是先学会把一个活动想清楚。需要定义：

- 活动目标：为什么要做这场 Space？解决什么学习、社区或传播问题？
- 目标听众：面向新人、开发者、研究者、项目方，还是课程学员？
- 核心主题：一句话说清活动主题，避免标题很大但讨论发散。
- 关键动作：希望听众完成什么动作？报名、入群、提问、提交作业、关注项目、参与 Week 3 组队？
- 活动边界：本次只做线上 Space / AMA / 小型分享，不追求复杂活动落地。
- 基础闭环：预热 → 参与 → 互动 → 反馈 → 复盘。

### 📚 推荐资料

Web3 实习手册｜社区运营指南、公开 Space / AMA 案例、Monad X、Base X、Figma / Canva、Notion / Google Sheet 排期表。

### 🛠 今日任务

- **任务 10｜Space / 小型活动策划 Brief**：选择一个 Web3 主题或学习场景，写清活动目标、目标听众、核心问题、嘉宾 / 主持角色、听众关键动作和风险边界。+20

## DAY 3｜路径深入：学习本方向真正的 Workflow

### 🎯 学习目标

学习如何把一个 Space 主题拆成议程、问题线、嘉宾沟通和宣传内容。

### 📖 学习内容

Day 3 的 Ops 重点是把“想办一场 Space”变成主持人和执行者真的可以拿来用的活动结构。需要学习：

- 主题定位：标题是否清楚？是否能让目标听众知道为什么要来？
- 活动简介：用 100–200 字说明活动背景、讨论问题和适合谁听。
- 嘉宾 / 主持分工：谁开场、谁控场、谁负责嘉宾沟通、谁记录问题和复盘？
- 问题线设计：准备 3–5 个核心问题，从基础问题到深入问题，避免一开始就太技术或太抽象。
- Run of Show：开场、嘉宾介绍、主题讨论、听众互动、CTA、收尾。
- 宣传内容：至少写 2 条不同角度的宣传文案，例如正式版和社群口语版。

**Space Run of Show 建议模板**

| 时间 | 内容 |
|------|------|
| 0–3 min | 开场：介绍活动主题、适合人群、互动方式 |
| 3–8 min | 嘉宾 / 主持介绍：说明嘉宾背景与本场讨论关系 |
| 8–25 min | 核心问题 1–2：从基础背景和真实经验进入 |
| 25–45 min | 核心问题 3–5：进入实践建议、工具、案例或争议 |
| 45–55 min | 听众互动：收集问题、邀请发言、回应评论区 |
| 55–60 min | 收尾 CTA：引导入群、提交作业、报名活动、查看资料或参与 Week 3 组队 |

### 📚 推荐资料

Web3 实习手册｜社区运营指南、中文排版规范、公开 Space / AMA 案例、X 宣传文案、Figma / Canva 素材工具。

## DAY 4｜路径产出：完成第一个 Proof of Work

### 🎯 学习目标

把 Space 策划变成一个可执行的活动运营方案。

### 📖 学习内容

Day 4 的 Ops 重点是从“议程”走向“执行”。一份合格的执行方案，应该让另一个同学拿到后也能知道怎么推进。需要包含：

- 活动基础信息：标题、时间、时长、平台、目标听众、活动形式。
- 角色分工：主持、嘉宾沟通、宣传、社群提醒、物料、记录、复盘。
- 宣传节奏：预热文案、当天提醒、开场前提醒、结束后回顾。
- 物料清单：海报 / banner、X 文案、TG / Discord 文案、问题收集表、复盘表。
- 互动机制：听众提问、评论区收集、举手发言、活动后表单。
- 数据指标：报名 / 预约人数、实际收听人数、互动问题数、入群 / 提交 / 关注等关键动作。
- 风险预案：嘉宾迟到、冷场、话题跑偏、技术故障、敏感问题、时间超出。
- 反馈闭环：活动后如何收集反馈、如何分类、如何回传给课程或项目团队。

### 📚 推荐资料

Web3 实习手册社区运营指南、公开 Space / AMA 案例、Notion / Google Sheet 排期表、Figma / Canva 活动物料、TG / Discord 社群管理工具。

### 🛠 今日任务

- **任务 12｜Space 执行预案**：设计一场 30–60 分钟 Space / AMA / 小型线上活动的执行方案，包含目标、受众、议程、嘉宾 / 主持、宣传节奏、分工、互动机制、预期指标和风险预案。+40
- **任务 13｜反馈与复盘框架设计**：在活动尚未执行的前提下，设计活动后要如何收集反馈、记录数据和复盘改进；提交反馈问题、记录表字段、复盘维度和改进回传方式，不要求提交真实数据。+30

### 📦 当日产出
