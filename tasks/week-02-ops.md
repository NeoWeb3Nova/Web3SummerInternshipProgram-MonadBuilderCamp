# Week 2 Ops Track: 运营、增长与生态协作

## 本周目标

完成 Monad 生态调研与用户观察，设计并执行一个最小化增长 / 教育 / onboarding 实验。

开营仪式提醒：Ops 方向不仅是发内容，也包括组织活动、表达复杂概念、做用户增长、收集反馈，并帮助新人更顺畅进入 Web3。

## Day-by-Day 任务

### Day 8: Monad 生态地图

- [ ] 识别 Monad 生态中的 5 个代表项目或工具
- [ ] 记录每个项目的：名称、定位、Twitter/X、Discord、网站、目标用户
- [ ] 在 `submissions/week-02-ops/ecosystem-map.md` 输出生态地图

### Day 9: 用户观察

- [ ] 加入 2-3 个 Monad / Web3 相关社群
- [ ] 记录 10+ 条用户讨论 / 痛点 / 问题
- [ ] 特别关注新人卡点：钱包、RPC、水龙头、部署、术语理解、任务打卡
- [ ] 整理到 `user-insights.md`

### Day 10: Campaign 设计

- [ ] 选择一个目标：新人 onboarding、Monad 高吞吐解释、Hackathon 组队、工具教程等
- [ ] 设计 Campaign 主题与 Hook
- [ ] 制定内容日历与渠道
- [ ] 写 `campaign-plan.md`

### Day 11: 内容制作

- [ ] 制作 3 条以上内容：短帖、图解、教程、FAQ、meme 任意组合
- [ ] 用 AI 辅助撰写，但保留人工判断和事实核验
- [ ] 保存草稿到 `assets/week-02-ops/`

### Day 12: 最小实验

- [ ] 在至少 1 个渠道发布或小范围测试
- [ ] 记录发布时间、平台、目标人群、互动数据
- [ ] 收集最早用户反馈

### Day 13: 反馈与迭代

- [ ] 分析曝光、互动、转发、评论、私聊反馈
- [ ] 找出最有效的一条内容或活动动作
- [ ] 调整下一轮内容
- [ ] 更新 Campaign 方案

### Day 14: 复盘与提交

- [ ] 整理增长数据
- [ ] 写运营复盘：目标、动作、结果、经验、下一步
- [ ] 准备 Week 3 组队时可贡献的用户洞察或叙事材料
