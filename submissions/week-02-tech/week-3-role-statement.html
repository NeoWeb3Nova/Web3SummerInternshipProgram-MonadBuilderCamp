我能承担什么角色

进入 Week 3 团队后，我希望承担 Tech / Core Dev 角色：把团队的问题定义快速收敛为可运行的技术方案，并负责智能合约、测试、部署脚本，以及前端与链上交互的端到端实现。我也可以承担技术验证与安全验证工作，先用最小实验判断协议、Agent 工具或交互路径是否可行，再决定是否投入完整开发，避免团队在错误方向上消耗时间。

我需要什么队友

我需要两类互补队友：一位擅长用户研究、产品定义或 Research 的队友，负责确认真实场景、核心用户和需求优先级；一位擅长视觉表达、运营或 Demo 叙事的队友，负责界面体验、演示节奏、用户反馈与对外表达。希望大家能直接沟通、快速决策，并接受用测试和用户反馈修正方案，而不是只停留在讨论阶段。

我能提供什么 Proof of Work

我能提供的 Proof of Work 不只是代码截图，而是一条可复验的证据链：公开 GitHub Repo、可运行源码、Foundry 或 Vitest 测试结果、部署脚本、Monad 测试网合约地址与交易哈希、关键调试记录、README、演示页面和短视频。本周我已经完成 Moss 的本地构建与测试，跑通 discover → load → action → simulate 流程，阅读核心模块，并实现可编译、可测试的 MockVault 协议适配器草稿及学习笔记。进入团队后，我会继续以小步提交、持续验证和透明记录推进协作，让每项贡献都能被队友复现、审查和继续开发。
