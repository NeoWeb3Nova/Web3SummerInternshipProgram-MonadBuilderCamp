# Submissions

这个目录存放每周和每个阶段的正式提交物。目标不是“填表”，而是把学习过程转成可验证、可复用、可展示的职业证据。

## 目录

- `week-01/` - 认知奠基期：钱包、链上体验、AI 工具、Monad 基础
- `week-02-tech/` - Tech Track：合约、前端、部署、测试
- `week-02-ops/` - Ops Track：生态地图、用户观察、Campaign、增长复盘
- `week-02-research/` - Research Track：研究大纲、分析文章、生态地图
- `week-03/` - 合作实战期：团队、问题定义、方案设计、Mini Demo
- `week-04-hackathon/` - Monad Builder Hackathon 提交
- `week-05-portfolio/` - 作品集、简历、面试、后续机会

## 提交规范

1. 每周末将交付物放入对应目录
2. 使用清晰的文件名，优先 Markdown / PDF / 链接记录
3. 在对应目录的 `README.md` 或清单文件中列出本周提交物
4. 保持 AI 协作透明性：记录 AI 做了什么、人做了什么判断
5. 尽量附上可验证证据：交易哈希、合约地址、Repo、截图、Demo 链接、文章链接
6. 每次提交都写一个简短复盘：目标、动作、结果、问题、下一步
